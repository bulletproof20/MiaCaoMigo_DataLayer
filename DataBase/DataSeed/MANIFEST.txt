MiaCaoMigo — DataSeed layer
============================

ROLE
----
Deterministic data tiers. Executable tests: DataBase/Tests/

DataSeed contains DATASETS ONLY (no PASS/FAIL assertions).

TIERS
-----
00_MasterData       System invariants — TRUNCATE + bootstrap rows
01_TestData         QA fixtures / stress — manual only (never default init)
02_DevelopmentData  Dev comfort set — manual / init_dev profile only
03_DemoData         Demo narrative — official bootstrap (after Master)

LOADERS (from DataBase/DataSeed)
--------------------------------
04_Loaders/00_MasterData.sql        Official bootstrap step 1
04_Loaders/01_DemoData.sql          Official bootstrap step 2
04_Loaders/02_DevelopmentData.sql   Manual dev / init_dev profile
04_Loaders/03_TestData.sql          Manual QA / init_test profile

BOOTSTRAP INTEGRATION
---------------------
  Bootstrap/init.sql → Profiles/init_demo.sql
    → Loaders/10_Official_Bootstrap.sql
      → 11_MasterData + 12_DemoData (delegates here)
    → Loaders/07_Sanity_Check.sql

  NOT in default init: TestData, DevelopmentData

PROFILES (Bootstrap/Profiles/)
-------------------------------
  init_minimal.sql   — no data
  init_master.sql    — Master only
  init_demo.sql      — Master + Demo (Docker default)
  init_dev.sql       — Master + Development
  init_test.sql      — Master + TestData
  init_full_qa.sql   — init_test + run Tests/runners/

QUICK START — FRESH DOCKER
--------------------------
  docker compose down -v
  docker compose up --build -d

QUICK START — QA (after init)
-----------------------------
  cd DataBase/Tests/runners
  .\run_full_qa.ps1

TIER COMPARISON
---------------
MasterData        Required invariants; TRUNCATE first; official bootstrap
DemoData          Staging narrative; append; official bootstrap
DevelopmentData   Small dev set; append; init_dev / manual only
TestData          Stress/fixtures; init_test / manual only
