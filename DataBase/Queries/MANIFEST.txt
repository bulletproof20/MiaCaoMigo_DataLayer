MiaCaoMigo — SQL Queries folder (reference only)
================================================

ROLE
----
Ad-hoc SELECT examples, service call catalogs, and API integration snippets.
NOT loaded by Docker init (see Bootstrap/init.sql).

SOURCE OF TRUTH
---------------
Operational logic lives in:
  - Schema/01_Modules/     DDL, views, procedures, jobs
  - Services/              Application PL/pgSQL (loaded via 06_Services.sql)

LAYOUT
------
00_Core/           Normalization and utility references
01_Module1/
  Authentication/  login_user / logout_user scenarios
  User_Creation/   fn_create_* validation matrix (00_RUN_ORDER.txt)
02_Module2/        Animal exploration SELECTs
03_Module3/        Commercial / inventory reference
04_Module4/        Appointment API-oriented SELECTs

DEDUPLICATION
-------------
Do not duplicate Services/00_Core helpers here.

USAGE
-----
Run manually against a database with Schema + Services applied.
Prefer fn_* / sp_* from Services over inline copies.

TESTS
